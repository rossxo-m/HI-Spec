# HI-Spec — Implementation Plan 03: GUI

**Scope:** Tasks 8–14. Build the single-card chrome-goo UI that matches `mockup/hi-spec-mockup.html`. Component-by-component, each with a headless snapshot test.

**Prereqs:** Plans 01 + 02 complete. APVTS stable, Graph processing audio.

**Visual reference:** `mockup/hi-spec-mockup.html` served at `http://localhost:5757`. Do not re-interpret — match it.

**Style:** Each component file <300 LOC. Each commit self-contained and runnable.

## Task 8 — LookAndFeel_HISpec

### Goal
One custom LNF for the whole plugin. Paints chrome bezels, Aqua gel buttons, frosted-aero panels, mercury text, and a reusable SVG-like goo filter effect for the spectrum visualiser.

### Files
- `Source/GUI/LookAndFeel_HISpec.h`
- `Source/GUI/LookAndFeel_HISpec.cpp`
- `Source/GUI/Assets/chrome_gradient.cpp` — palette + gradient helpers
- `tests/snapshot_lookandfeel.cpp`

### Design notes

**Palettes.** Three named gradients baked as `juce::ColourGradient` factories:
- `chromeBezel()` — 9-stop conic approximation: `#f8f8f8 → #d0d0d0 → #7a7a7a → #efefef → #9a9a9a → #cfcfcf → #6d6d6d → #e8e8e8 → #f8f8f8`.
- `aquaGel(Colour accent)` — radial from `accent.brighter(0.8)` to `accent` with specular top highlight.
- `frostedAero()` — `Colours::white.withAlpha(0.55f)` over a noise dither layer.

**Knobs.** Override `drawRotarySlider`: chrome ring, inky pointer, tiny value readout below (hidden by default, shown on hover). No bitmap cache in v1 — profile later.

**Goo filter.** JUCE has no SVG filter equivalent. Approximate: render source to a temp `Image`, Gaussian blur with radius ~6px, threshold alpha via a `ColourMatrixEffect`-style LUT (`alpha<0.5 → 0`, `>=0.5 → 1`). Package as `GooFilterEffect : juce::ImageEffectFilter`.

### Tests
- `snapshot_lookandfeel.cpp::"chrome bezel renders 9 bands"` — render a 400×80 rect, sample 9 horizontal midpoints, assert luminance follows the defined curve (±5%).
- `"aqua gel has specular highlight on top third"` — render button, top-third mean luminance > bottom-third by ≥0.2.
- `"goo filter merges two close discs into one blob"` — draw two circles 4px apart, apply filter, flood-fill from one, expect the other is reached.

### Commit
`feat(gui): LookAndFeel_HISpec with chrome, aqua, goo filter`

## Task 9 — SpectrumView

### Goal
Full-width top card. Live 32-band spectrum bars with chrome-goo visual language. Click-to-select-band.

### Files
- `Source/GUI/SpectrumView.h`
- `Source/GUI/SpectrumView.cpp`
- `tests/snapshot_spectrum.cpp`

### Design notes

Subscribes to a lock-free `AbstractFifo` filled by `PluginProcessor` with per-band magnitudes (one snapshot per ~30Hz frame). Paints bars with a goo-filtered gradient. Highlighted band (if any) gets a bright mercury rim.

`onBandClicked(int bandIndex)` is a `std::function` set by the editor; the editor forwards to `selectedBand` state.

### Tests
- `"renders 8 bars in 8-band mode"` — inject 8 magnitudes, count distinct bar centroids in snapshot.
- `"selected band is visibly brighter"` — with band 5 selected, its bar's mean luminance > median of others by ≥0.15.
- `"click at bar 3 x-center fires onBandClicked(3)"`.

### Commit
`feat(gui): SpectrumView with goo bars and click-select`

## Task 10 — BandDetailStrip

### Goal
Thin strip under the spectrum. Shows the currently selected band's per-band params: time, feedback, gain, pan, pitch shift. When no band is selected, shows "Global" aggregate readouts.

### Files
- `Source/GUI/BandDetailStrip.h`
- `Source/GUI/BandDetailStrip.cpp`
- `tests/snapshot_band_strip.cpp`

### Design notes

Five small chrome knobs + labels. Knobs attach to `delay_band<N>_*` params when a band is selected, detach when deselected. No re-layout on selection change — labels just repurpose.

### Tests
- `"attaching to band 2 binds five params"` — select band 2, programmatically change each knob, assert matching `delay_band2_*` values updated.
- `"deselecting detaches cleanly"` — no param callbacks fire after deselection.

### Commit
`feat(gui): BandDetailStrip with per-band knob binding`

## Task 11 — FxModuleCard + PositionChip

### Goal
The reusable card chrome that both Vocoder and SpectralFilter bodies live inside. Owns the power LED, name, **Position chip** (5-cell matrix dropdown), per-module Mix knob, and a content slot for the body.

### Files
- `Source/GUI/FxModuleCard.h`
- `Source/GUI/FxModuleCard.cpp`
- `Source/GUI/PositionChip.h`
- `Source/GUI/PositionChip.cpp`
- `tests/snapshot_fx_card.cpp`
- `tests/test_position_chip.cpp`

### Design notes

**PositionChip** is a small aqua-gel button. Click opens a 3×2 grid popup: rows `{Pre, Post, FB}`, cols `{Global, Per-Band}`. The `FB × Global` cell is disabled (FB is per-band-only — locked decision). Selection writes `{module}_position` enum param.

**FxModuleCard** takes `juce::Component& body` in ctor and a short color accent (violet for Vocoder, amber for SpectralFilter).

### Tests
- `test_position_chip.cpp::"FB_Global is disabled"` — try to set it programmatically, expect rejection + no change.
- `"cycles through 5 valid cells"` — iterate, assert all 5 reachable, 6th click wraps.
- `snapshot_fx_card.cpp::"violet accent vs amber accent differ in hue"` — render both, sample card header luminance/hue.

### Commit
`feat(gui): FxModuleCard chrome + PositionChip 5-cell dropdown`

## Task 12 — VocoderBody

### Goal
The inside of the Vocoder card. Matches mockup: Carrier segmented (Self/Noise), vowel pad (4 corners A/E/I/O with draggable dot), Q + Shift knobs, Texture sub-section (Drive/Rate/Depth/Delay knobs + Sine/Tri/ThruZero segmented).

### Files
- `Source/GUI/VocoderBody.h`
- `Source/GUI/VocoderBody.cpp`
- `Source/GUI/VowelPad.h` / `.cpp`
- `tests/snapshot_vocoder.cpp`
- `tests/test_vowel_pad.cpp`

### Design notes

**VowelPad** is a 2D xy control. Corners labeled A/E/I/O. Output is a pair of normalized `(x, y)` → blended `(voc_formantX, voc_formantY)` params. Label fonts are mercury-style (chrome clip-text).

### Tests
- `test_vowel_pad.cpp::"corner A maps to (0,0)"` — click top-left, expect `voc_formantX=0 voc_formantY=0`.
- `"drag moves dot continuously"` — simulate 10-point drag, assert monotonic param change.
- `snapshot_vocoder.cpp::"Texture sub-section rendered below main row"` — snapshot, assert bounding box of Texture group sits below vowel pad bottom.

### Commit
`feat(gui): VocoderBody with vowel pad and Texture sub-section`

## Task 13 — SpectralFilterBody

### Goal
Inside of the Spectral Filter card. Shape segmented (LP/HP/BP/Notch), live response curve SVG-style plot, Freq + Q knobs, Mod sub-section (Rate/LFO-amt/Env-amt/Atk/Rel + Sine/Tri/S&H).

### Files
- `Source/GUI/SpectralFilterBody.h`
- `Source/GUI/SpectralFilterBody.cpp`
- `Source/GUI/FilterResponsePlot.h` / `.cpp`
- `tests/snapshot_filter.cpp`

### Design notes

**FilterResponsePlot** computes the magnitude response of the active shape at 128 log-spaced frequencies each frame, paints a chrome-goo polyline. Updates on param change (throttle to 30Hz).

### Tests
- `"LP shape has falling high end"` — set shape=LP freq=1kHz Q=0.707, sample response at 100Hz and 10kHz, expect 10kHz < 100Hz by ≥20 dB.
- `"BP shape peaks at center freq"` — argmax of 128 samples within ±1 bin of `sf_freq`.
- `snapshot_filter.cpp::"Mod sub-section rendered below curve"`.

### Commit
`feat(gui): SpectralFilterBody with live response plot and Mod section`

## Task 14 — HeaderBar + Breadcrumb + PluginEditor assembly

### Goal
Final assembly. Header (mercury logo, preset menu, Simple/Advanced toggle, A/B gel buttons), breadcrumb ("Spectrum / Band 12 / Vocoder"), then spectrum → band-strip → fx-row. Editor is 1100×700, non-resizable (resizable is a v2 candidate per density memo).

### Files
- `Source/GUI/HeaderBar.h` / `.cpp`
- `Source/GUI/Breadcrumb.h` / `.cpp`
- `Source/PluginEditor.h`
- `Source/PluginEditor.cpp`
- `tests/snapshot_full_editor.cpp`

### Design notes

**Simple/Advanced toggle** hides `.fx-row` + `.band-strip` (matches mockup JS behavior). Spectrum remains full-width and expands vertically to fill freed space.

**A/B state buttons** snapshot current APVTS state into two slots, swap on click. A third "copy A→B" micro-button clones.

**Preset menu** opens a `juce::PopupMenu` populated from `PresetManager::listFactoryPresets() + listUserPresets()`. Selecting fires `PresetManager::loadPreset`.

### Tests
- `snapshot_full_editor.cpp::"default layout at 1100×700"` — full-editor snapshot, assert spectrum, band-strip, fx-row bounding boxes match expected within ±4px of mockup reference.
- `"Simple mode collapses fx-row"` — toggle Simple, assert fx-row not visible, spectrum height grew.
- `"A/B swap flips Vocoder power"` — set A with voc_power=1, swap to B (empty), assert voc_power=0.

### Commit
`feat(gui): HeaderBar + Breadcrumb + PluginEditor full assembly`
